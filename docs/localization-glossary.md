# Timviz localization glossary

Timviz is an appointment and service-booking product. Client-facing text should sound like booking time with a specialist, not hotel reservations or shopping orders.

## Global rules

- Use short UI labels for buttons and mobile controls.
- Keep one term for the same object inside one language.
- Avoid hotel-style reservation wording in client flows.
- Legal, SEO, and pro-dashboard copy may use local SaaS terms for "booking software" when that is the natural search phrase, but appointment wording is preferred in customer-facing flows.
- If a translation is uncertain, use the English fallback temporarily and mark it for native review.

## Avoid

- RU/UK: avoid "бронь", "бронювання" in client appointment UX; use "запись/запис".
- FR: avoid generic "réservation" in client appointment UX; use "rendez-vous" or short "RDV".
- PL: avoid overusing "rezerwacja" in client UX; use "wizyta" or "zapisy".
- CS: avoid "rezervace" for client CTA when "objednání/objednat se" is more natural.
- ES: avoid "reserva" in client appointment UX; use "cita".
- DE: avoid "Buchung" in client UX; use "Termin".
